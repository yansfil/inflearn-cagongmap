# 카공맵 MVP 범위 정리

> 노트북 작업하기 좋은 카페를 지도에서 찾는 서비스
> 최종 갱신: 2026-06-12

## 합의된 전제

리서치 보고서를 바탕으로 한 논의에서 확정된 방향:

| 항목 | 결정 |
|---|---|
| **1차 목표** | 가설 검증용 MVP (얇게, 빠르게) |
| **데이터 수급** | 운영자 직접 시드 (UGC는 나중) |
| **플랫폼** | 웹 (모바일 웹 우선) |
| **타겟 세그먼트** | 특정 안 함 (전체) — 단, 필터는 공통 1순위만 |

핵심 전략: UGC에만 의존하지 않고 **콘센트·와이파이·카공 허용 3대 핵심 데이터를 운영자가 시드로 구축**한 뒤, **단일 권역**부터 시작한다.

---

## ✅ 이번 구현에서 먼저 다룰 기능

1. **지도 기반 카페 탐색** — 모바일 웹, 지도 위에 시드 카페 핀 표시. 단일 권역(강남·홍대·성수 중 1곳)으로 한정.
2. **3대 핵심 필터** — 콘센트(등급), 와이파이, 카공 허용 여부. 모든 세그먼트의 공통 1순위 교집합.
   - 직장인 1순위 = 콘센트(하드웨어), 학생 1순위 = 카공 허용(눈치). 이 셋은 세그먼트 교집합이라 타겟을 좁히지 않고도 차별점을 확보한다.
3. **카페 상세 카드** — 핀 클릭 시 3대 속성 + 영업시간 + 사진 1~2장 + "최근 확인됨" 타임스탬프.
4. **필터링 / 정렬** — 3대 속성으로 핀을 거르고, 리스트 뷰로도 열람.
5. **운영자 시드 데이터 입력 구조** — UGC 없이 운영자가 50~100개를 채울 수 있는 데이터 스키마 + 입력 방식(간단한 관리 화면 또는 시트→정적 JSON).

---

## ⏸️ 지금은 미뤄도 되는 기능

- **실시간 혼잡도 / 좌석 정보** — 데이터 소스 없음, 구현 난이도 높음. 가설 검증 단계에선 과함.
- **UGC(사용자 제보 / 등록)** — 신선도 루프는 2단계. MVP는 운영자 시드로 충분.
- **제보 인센티브(포인트·뱃지), 신고 / 검증 기능**
- **회원가입 / 로그인** — 시드 DB 열람만이면 불필요. 즐겨찾기는 로컬스토리지로 대체 가능.
- **수익화(광고·송객 제휴)** — 트래픽 임계량 도달 후.
- **권역 확장, 네이티브 앱, 푸시 알림**
- **부가 필터(1인석·소음·조명·가격대 등)** — 핵심 가설 검증 후 추가 여부 판단. (아래 미확정 이슈 ①)

---

## 📍 필요한 장소 데이터 (카페당 시드 스키마)

| 필드 | 타입 | 비고 |
|---|---|---|
| 이름 / 주소 / 좌표(lat,lng) | 필수 | 지도 핀 |
| 콘센트 등급 | enum (많음 / 보통 / 적음 / 없음) | 핵심 필터 1 |
| 와이파이 | enum (있음·안정 / 있음 / 없음) | 핵심 필터 2 |
| 카공 허용 | enum (환영 / 허용 / 눈치 / 금지) | 핵심 필터 3 — 가장 차별적 |
| 영업시간 | text | 심야 / 24h 여부 판단 |
| 사진 | 1~2장 URL | 분위기 파악 |
| 최근 확인일 | date | 신선도 표시 |
| (선택) 1인석 / 좌석 규모 / 가격대 | enum | 미확정 이슈 ①에 따라 포함 여부 결정 |

→ 좌표·주소·영업시간은 지도 API나 공개 정보로 보강 가능하지만, **콘센트·와이파이·카공 허용 3대 속성은 운영자가 직접 방문 / 리뷰 조사로 채워야 하는 핵심 자산**이다.

---

## ⚠️ 사용자(운영자)가 직접 확인해야 할 가정

1. **권역 선택** — 강남 / 홍대 / 성수 중 어디로?
   - 성수 = 트렌디·작업카페 많음 / 강남 = 직장인·취준 / 홍대 = 학생. 사용자층·카페 밀도·카공 분위기가 다르다.
2. **시드 데이터 수집 방법의 현실성** — 50~100개의 콘센트·카공 허용 정보를 *어떻게* 채울지. 직접 방문은 비현실적일 수 있고, 블로그 / 리뷰 크롤링은 신뢰성 문제. **MVP 최대 병목.**
3. **"카공 허용" 판정 기준** — 누가 어떤 근거로 "환영 / 눈치 / 금지"를 정하나? 매장 공지? 리뷰 분위기? 운영자 주관? 이 라벨의 신뢰도가 서비스 신뢰도를 좌우한다.
4. **지도 SDK 선택** — 카카오맵 / 네이버지도 / Google Maps 중 무엇? 국내 카페 POI는 카카오 / 네이버가 유리. 사용 약관·쿼터 확인 필요.

## ❓ 미확정 이슈 (다음 논의에서 결정)

① **필터 범위** — 콘센트·와이파이·카공 허용 3개만 둘지, 1인석·영업시간까지 필터로 넣을지.
   - 현재 제안(기본값): **3대 핵심 필터만.** 가설 검증 MVP에서 필터를 다 넣으면 "이 도구가 왜 좋은지"가 흐려진다.

② **검증하려는 핵심 가설** — MVP 성패 지표를 정하는 기준.
   - 현재 제안(기본값): **"기존 방법(지도+블로그 교차검색)이 불편해서, 작업 특화 속성으로 필터링되는 전용 지도가 있으면 쓴다."**
   - 대안: "혼잡도 / 실시간성"이 진짜 핵심 가설일 수도 있음 → 이 경우 미뤄둔 실시간 기능을 앞당겨야 함.
